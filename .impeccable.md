## Design Context

### Users

Linux.do readers use this Tampermonkey sidebar while reading long discussion topics. Their primary jobs are to summarize a bounded floor range, inspect service-returned reasoning without confusing it with the final answer, quote a useful summary fragment into follow-up chat, and manage a streaming reply without leaving the topic page.

### Brand Personality

Calm, precise, trustworthy. The interface should feel like a focused reading and analysis tool: capable without looking experimental, restrained without becoming anonymous, and explicit about request state and destructive actions.

### Aesthetic Direction

The default Style2 theme uses an independently implemented Cerebr-inspired visual language: warm tinted neutrals, a restrained steel-blue accent, compact editorial typography, shallow elevation, and clear light/dark modes. Avoid the previous black-and-gold emphasis, neon or purple-blue AI gradients, gradient text, decorative emoji, heavy glass blur, elastic motion, and card-within-card layouts. Style1 remains a compatibility theme and is not visually redesigned.

### Design Principles

1. Make request state and the next safe action immediately legible.
2. Prefer semantic controls, keyboard parity, visible focus, and touch targets of at least 40px.
3. Keep reasoning, final answers, status metadata, and copied text strictly separated.
4. Preserve the attached desktop sidebar, but switch to a non-squeezing overlay below the narrow-screen breakpoint.
5. Use a small semantic token system and purposeful motion; add no remote visual assets or runtime UI dependencies.
